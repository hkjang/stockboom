# 사용자 관리 가이드 (User Management Guide)

## 1. 개요
본 문서는 관리자가 시스템 사용자의 계정, 권한, 증권사 연동 상태를 관리하는 절차를 설명합니다.

## 2. 계정 관리

### 2.1 사용자 조회 및 상태 변경
- **목록 조회**: 가입일, 최근 접속일, 이메일 인증 여부, 활성 상태 필터링
- **상태 변경**:
  - **활성(Active)**: 정상 이용 가능
  - **비활성(Inactive)**: 로그인 차단 (약관 위반, 요청 등)
  - **잠김(Locked)**: 로그인 실패 횟수 초과 (관리자 해제 필요)
- **계정 삭제**: 사용자 요청 시 개인정보 파기 절차에 따라 삭제 (데이터 보존 기간 준수)

### 2.2 권한(Role) 관리
- **User**: 일반 사용자 (기본)
- **Admin**: 시스템 관리자 (모든 메뉴 접근 가능)
- **Manager**: 운영자 (일부 관리 메뉴 접근 가능)
- **권한 부여**: 사용자 상세 페이지에서 Role 변경 가능

## 3. 증권사 API 연동 관리

### 3.1 API Key 등록 및 갱신 지원
- 사용자가 직접 등록하기 어려운 경우 관리자가 대행 등록 (보안 주의)
- **Key 만료 관리**: 만료 임박 계정 조회 및 알림 발송 (이메일/SMS)
- **오류 계정 관리**: 연동 실패(인증 오류 등) 계정 식별 및 사용자 안내

### 3.2 모의투자 vs 실전투자
- 사용자별 모의투자/실전투자 모드 확인
- 실전투자 전환 승인 절차 (필요 시)

## 4. 매매 전략 승인

### 4.1 전략 심사
- 사용자가 생성한 커스텀 전략의 로직 검토
- **위험성 평가**: 과도한 레버리지, 빈번한 매매 등 위험 요소 확인
- **승인/거부**: 심사 결과에 따라 전략 활성화 승인 또는 거부 사유 통보

### 4.2 시스템 전략 배포
- 관리자가 생성한 공통 전략(추천 전략)을 전체 또는 특정 그룹 사용자에게 배포
- 전략 성과 모니터링 및 파라미터 튜닝
